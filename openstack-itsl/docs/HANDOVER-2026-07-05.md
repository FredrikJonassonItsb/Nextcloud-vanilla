## 3. Vad som är deployat + hur man opererar

**Åtkomst dev15:** `ssh ubuntu@10.43.51.62` (kräver ITSL-nät/VPN). occ:
`sudo docker exec -u www-data hubs-php php /var/www/html/occ <cmd>`. Postgres (Hubs):
`sudo docker exec hubs-postgres psql -U oc_hubs -d hubs`. Openstack-db:
`sudo docker exec openstack-brain-db-1 psql -U postgres -d <brain_x|engine_meta>`.

**Deploy-mönster (från Windows-host, Git Bash):**
- Compose-stacken: `cd openstack-itsl && bash stack/deploy.sh` (tar-over-ssh → /opt/openstack, cert-autodetect, up -d --build).
- agent_engine-appen: `bash provision/deploy-app.sh` (tar → hubs-php custom_apps, occ app:enable/upgrade, verifierar OCS 401).
- Enskild fil snabbt: `MSYS_NO_PATHCONV=1 tar czf - -C apps agent_engine/lib/X.php | ssh ... 'docker exec -i hubs-php tar xzf - -C /var/www/html/custom_apps && chown www-data ...'` (opcache plockar upp; ingen restart).
- Provisionering: `bash provision/occ-provision.sh` (idempotent). Deck-tavla: `node provision/deck-bootstrap.mjs --humans <uid,...>`. Deck-patch: `bash patches/apply-deck-patches.sh`.

**Nycklar/secrets:** bara i `/opt/openstack/.env` (chmod 600) — ALDRIG i repot. `.env`/`.env.test`/`state/`/node_modules/vendor är gitignore:ade. Register i [docs/SECRETS-TRACKER.md](SECRETS-TRACKER.md). Anthropic + OpenRouter-nycklar satta (OpenRouter-kontot har krediter → embeddings live).

## 8. KRITISKA FÄLLOR nästa session MÅSTE känna till

- ⚠️ **Kör ALDRIG `docker restart hubs-php`** — NC:s entrypoint rensar `apps/sdkmc`-tillägg (se [[hubs-ops]]). opcache `validate_timestamps` är PÅ → filändringar plockas upp utan restart.
- **NC Entity-fällor:** (a) `setter()` skriver inte oförändrade värden → ge NOT NULL-kolumner DB-default; (b) `method_exists()` ger FALSE för magiska `__call`-getters → anropa direkt / `is_callable`.
- **Numeriska BankID-uid** (personnummer) bryter mot naiva strikta jämförelser och personnummer-regexer — uid får ALDRIG hamna i agentläsbar text eller firewall-skannade fält; håll dem i strukturerade metadatafält (uid-i-text-regeln, DEV15-FACTS).
- **Nästlade heredocs** i `ssh bash -s <<'REMOTE'` med inre `<<'SQL'` fungerar INTE tillförlitligt — använd direkta `psql -c` eller separata anrop.
- **ssh "cmd"** kör under dash (ingen `pipefail`) — använd `ssh ... bash -s <<<"$script"`.
- **Windows/Git Bash:** `python`/`python3` är Store-shim (saknas ofta) — parsa JSON med `node`/`grep` lokalt, python3 finns på servern. `MSYS_NO_PATHCONV=1` för tar/ssh-paths. LF→CRLF-varningar vid git add är ofarliga. `UID` är readonly i bash (döp om).
- **Git Bash skickar multibyte-tecken fel till curl** — skriv body till UTF-8-fil + `--data-binary @fil` (lib.sh:s `http()` gör detta).
- **Deck 1.15.9:** ingen assign-event; `assignUser`/`unassignUser` dispatchar `CardUpdatedEvent`. Deck-event bär ej assignedUsers → hämta kortet färskt. reorder-routen tar target-stack i URL-pathen.
- **agent_engine deploy** kräver att OCS-rutten svarar 401 (ej 404) efter enable — deploy-app.sh verifierar.
- Deck-patchen (`patches/`) måste köras om efter varje Deck-uppdatering.
