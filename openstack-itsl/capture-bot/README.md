# capture-bot

Talk capture bot per CONTRACTS §6. Node 22, single service, port **8790**, non-root container.

Flow for `POST /bot` (Talk bot webhook):

1. Route the room token (from the payload) via `rooms.json` → `{ brain, botEnv, mode }`. Unrouted room ⇒ 200 ignore.
2. Verify `X-Nextcloud-Talk-Signature` = HMAC-SHA256(`TALK_BOT_SECRET_<botEnv>`, `X-Nextcloud-Talk-Random` + raw body). Fail ⇒ 401.
3. Skip non-`Create` events, bot-authored and empty messages.
4. Dedupe: atomic claim in `engine_meta.capture_seen` (`DATABASE_URL`; table auto-created). Duplicate ⇒ 200, no side effects.
5. PII firewall (`pii-patterns.json`, shared list — CONTRACTS §3) **before anything leaves the house** ⇒ 422 + threaded `Blockerat: …` reply. Fails **closed** if the pattern file cannot load.
6. `POST {brain}/ingest` with `Authorization: Bearer BRAIN_KEY_<NAME>`; metadata `{source:"talk", talk_id, talk_room, talk_actor}`; team room additionally `author` = speaker uid (structural attribution). Brain 422 ⇒ `Blockerat: <reason>`.
7. **Ingest before reply.** Only after a stored capture: 👍 reaction + threaded `Sparat i <hjärna> ✓` (HMAC-signed bot reply API). Reply failures are logged, never fatal, never lose a capture. Transient ingest failure ⇒ claim released + 500 so Talk retries.

Commands (only grammar allowed, INTERAKTIONSDESIGN §3):

- `!queue <text>` — captures as above **and** creates a Deck card in the `Inbox` stack on the *Agent Engine* board as `bot-engine` (Basic auth, `BOT_APP_PASSWORD_ENGINE`): title `[inbox][<agentkod>] <first line>`, description = full message, label `needs-enrichment` (resolve-or-create), assignee = sender. Replies with the card link. Board/stack ids from `bootstrap.json`.
- `!status [agent]` — reads `GET /ocs/v2.php/apps/agent_engine/api/v1/ledger/{agentCode}` as `bot-engine` and replies with a compact Swedish status line. Defaults to the sender's own agent (CONTRACTS §1 identity map), else the room's brain.

`GET /healthz` ⇒ `{ ok: true, dbReady }`.

## Configuration (env)

| Var | Meaning |
|---|---|
| `PORT` | default 8790 |
| `NC_BASE` | e.g. `https://dev15.hubs.se` |
| `DATABASE_URL` | postgres URL to **engine_meta** as `svc_engine` |
| `TALK_BOT_SECRET_REB\|ATLAS\|ADA\|MARVIN` | per-bot Talk secrets (same as `occ talk:bot:install`) |
| `BRAIN_KEY_REB\|ATLAS\|ADA\|MARVIN\|TEAM` | bearer keys for `/ingest` |
| `BRAIN_URL_<NAME>` | override brain base URL (default `http://brain-<name>:710X` per CONTRACTS §4) |
| `BOT_APP_PASSWORD_ENGINE` | app password for `bot-engine` (Deck + agent_engine) |
| `BOT_ENGINE_USER` | default `bot-engine` |
| `ROOMS_PATH` | default `./rooms.json` (mounted; generated by occ-provision) |
| `PII_PATTERNS_PATH` | default bundled copy; mount `stack/shared/pii-patterns.json` over it |
| `BOOTSTRAP_PATH` | default `/opt/openstack/state/bootstrap.json` (read-only mount) |

## rooms.json

Generated by `provision/occ-provision.sh` from `rooms.json.template`:

```json
{ "<roomToken>": { "brain": "reb|atlas|ada|marvin|team", "botEnv": "REB|ATLAS|ADA|MARVIN", "mode": "personal|team" } }
```

Exactly one bot is set up per room; `botEnv` selects both the inbound verification secret and the outbound signing secret.

## bootstrap.json (expected shape)

Written by deck-bootstrap. Accepted shapes: `{"board":{"id":N},"stacks":{"Inbox":M,…}}`, or `boardId`/`board_id` top-level, or `stacks` as an array of `{id,title}`. If the Inbox stack id is absent it is resolved live via `GET /boards/{id}/stacks`.
